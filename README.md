# Rustphy

A programming language implementation built in Rust for learning compiler design and implementation techniques.

## Overview

Rustphy is an educational programming language project that demonstrates the complete compilation pipeline from source code to executable formats. It features a clean, JavaScript-like syntax and multiple compilation targets for exploring different execution strategies.

## Features

### 🔍 Lexer

- Tokenizes source code into a stream of tokens
- Supports identifiers, integers, strings, operators, and keywords
- Handles compound assignment operators (`+=`, `-=`, `*=`, `/=`)

### 🌳 Parser

- Recursive descent parser with Pratt parsing for expressions
- Generates an Abstract Syntax Tree (AST) from tokens
- Full JSON serialization support for AST inspection
- Two parser implementations available (`parser` and `parser_v2`)

### ⚙️ Bytecode Compiler

- Compiles AST to custom bytecode instructions
- Stack-based virtual machine architecture
- Includes symbol table for variable resolution
- Runtime environment with scope management

### 🌐 WebAssembly Compiler (In Progress)

- Direct compilation to WebAssembly Text (WAT) format
- Two compiler approaches:
  - `WasmCompiler`: Full-featured WASM compilation
  - `WasmSimpleCompiler`: Simplified WASM output
- WASM bindings for browser execution via wasm-bindgen

## Getting Started

### Quick Start in 5 Minutes

1. **Install Rust** (if you haven't already)

   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```

2. **Clone and Build Rustphy**

   ```bash
   git clone <repository-url>
   cd rustphy
   cargo build --release
   ```

3. **Write Your First Program**

   Create a file called `hello.gph`:

   ```javascript
   let name = "World";
   print("Hello, " + name + "!");

   fn greet(person) {
       return "Welcome, " + person;
   }

   print(greet("Rustphy"));
   ```

4. **Run It!**

   ```bash
   ./target/release/rustphy run hello.gph
   ```

   Output:

   ```
   Hello, World!
   Welcome, Rustphy
   ```

### Step-by-Step Tutorial

#### 1️⃣ Variables and Arithmetic

Create `math.gph`:

```javascript
let x = 10;
let y = 20;
let sum = x + y;
let product = x * y;

print("Sum: " + sum);
print("Product: " + product);
```

Run:

```bash
rustphy run math.gph
```

#### 2️⃣ Functions and Recursion

Create `fibonacci.gph`:

```javascript
fn fib(n) {
    if (n <= 1) {
        return n;
    }
    return fib(n - 1) + fib(n - 2);
}

print("Fibonacci(10) = " + fib(10));
```

#### 3️⃣ Loops and Arrays

Create `arrays.gph`:

```javascript
let numbers = [5, 10, 15, 20, 25];
let total = 0;

for (let i = 0; i < 5; i += 1) {
  total += numbers[i];
  print("Added " + numbers[i] + ", total now: " + total);
}

print("Final sum: " + total);
```

#### 4️⃣ Exploring the Compiler Pipeline

**See how your code is parsed:**

```bash
rustphy parse hello.gph
```

This shows the Abstract Syntax Tree (AST) as JSON.

**See the generated bytecode:**

```bash
rustphy compile bytecode hello.gph
```

This displays the low-level instructions for the virtual machine.

**Generate WebAssembly:**

```bash
rustphy compile wasm hello.gph
```

This outputs WebAssembly Text (WAT) format.

### Interactive Mode

Use Rustphy with stdin for quick experiments:

```bash
# Quick calculations
echo 'print(10 + 20 * 3);' | rustphy run

# Test functions
echo 'fn double(x) { return x * 2; } print(double(21));' | rustphy run

# Inspect AST
echo 'let x = 5 + 10;' | rustphy parse

# Generate bytecode
echo 'let x = 10;' | rustphy compile bc
```

### Understanding the Output

When you run `rustphy compile bytecode program.gph`, you'll see output like:

```
0000 LoadConst(Int(10))
0001 DefineGlobal("x")
0002 GetGlobal("x")
0003 Print
```

Each line shows:

- **Index**: Instruction position
- **Opcode**: The operation to perform
- **Operand**: Data for the operation

This teaches you how high-level code becomes low-level instructions!

## Usage

### Run a Rustphy Program

```bash
# Run from file
rustphy run script.gph

# Run from stdin
echo 'let x = 5; print(x);' | rustphy run
```

### Parse and Inspect AST

```bash
# Parse and output AST as JSON
rustphy parse script.gph

# Or from stdin
echo 'let x = 10 + 5;' | rustphy parse
```

### Compile to Different Targets

```bash
# Compile to bytecode
rustphy compile bytecode script.gph
# Or shorthand
rustphy compile bc script.gph

# Compile to WebAssembly (WAT format)
rustphy compile wasm script.gph
# Or shorthand
rustphy compile wat script.gph
```

### CLI Commands

```
rustphy <COMMAND> [OPTIONS]

COMMANDS:
    run [FILE]              Run a Rustphy program
    parse [FILE]            Parse source code and print the AST as JSON
    compile <TARGET> [FILE] Compile to the specified target (bytecode|bc|wasm|wat)
    version                 Print version information
    help                    Print help message
```

## Language Syntax

Rustphy features a familiar C-style syntax with influences from JavaScript:

### Variables

```javascript
let x = 5;
let name = "rustphy";
let flag = true;
```

### Arithmetic and Operations

```javascript
let sum = 10 + 20;
let product = 5 * 4;
let result = (10 + 5) / 3;

// Compound assignments
x += 10;
x *= 2;
```

### Conditionals

```javascript
if (x > 10) {
  print("x is greater than 10");
} else {
  print("x is 10 or less");
}
```

### Loops

```javascript
for (let i = 0; i < 10; i += 1) {
  print(i);
}
```

### Functions

```javascript
fn add(a, b) {
    return a + b;
}

let result = add(5, 3);
```

### Arrays

```javascript
let numbers = [1, 2, 3, 4, 5];
let first = numbers[0];
```

### Boolean Operations

```javascript
let isValid = true && false;
let hasAccess = isAdmin || isModerator;
let isNotEmpty = !isEmpty;
```

## Architecture

The project is organized into several key modules:

```
src/
├── lexer.rs              # Tokenization
├── token.rs              # Token definitions
├── parser.rs             # Primary parser implementation
├── parser_v2.rs          # Alternative parser
├── ast.rs                # AST node definitions
├── bytecode_compiler.rs  # Bytecode generation
├── instruction.rs        # Bytecode instruction set
├── machine.rs            # Virtual machine execution
├── environment.rs        # Runtime environment and scoping
├── symbol_table.rs       # Compile-time symbol resolution
├── compiler.rs           # WASM compiler (full)
├── wasm_simple_compiler.rs # WASM compiler (simplified)
├── wasm_environment.rs   # WASM runtime helpers
├── lib.rs                # Public API and WASM bindings
└── bin/rustphy.rs        # CLI entry point
```

### Compilation Pipeline

1. **Source Code** → Lexer → **Tokens**
2. **Tokens** → Parser → **AST**
3. **AST** → Compiler → **Bytecode** or **WASM**
4. **Bytecode** → Virtual Machine → **Result**
5. **WASM** → Browser/Runtime → **Result**
